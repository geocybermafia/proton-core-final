# Proton-Core AI - Components & Routing Documentation

ეს დოკუმენტი აღწერს პროექტის ინტერფეისისა და სერვერული როუტინგის ბოლოდროინდელ მნიშვნელოვან ოპტიმიზაციებს, რათა მომავალში მარტივად შევძლოთ ნეირო-ვორქფლოუების სინქრონიზაცია.

## 1. Vercel Single Page Application (SPA) Routing — `vercel.json`
* **პრობლემა:** სხვადასხვა ბრაუზერებში (მათ შორის Opera, Chrome) ან მობილურ მოწყობილობებზე საიტის პირდაპირი ლინკით (Deep Linking) გახსნისას ან გვერდის რეფრეშისას სერვერი აბრუნებდა `404 NOT_FOUND` შეცდომას.
* **გადაწყვეტა:** პროექტის მთავარ დირექტორიაში (Root Folder) შეიქმნა `vercel.json` ფაილი შემდეგი Rewrite წესით:
  ```json
  {
    "rewrites": [
      { "source": "/(.*)", "destination": "/index.html" }
    ]
  }
  ```
  ეს უზრუნველყოფს იმას, რომ ვერსელის სერვერმა ყველა შიდა როუტი გადაამისამართოს `index.html`-თან, რის შემდეგაც კლიენტის მხარეს React Router უმტკივნეულოდ მართავს შესაბამის გვერდებსა და როუტის ანიმაციებს.

## 2. MarketView მობილური ადაპტაცია & ვიზუალური ფიქსები
* **პრობლემა:** ქარდების ქვედა ნაწილში სერვისის/პროდუქტის ბეიჯი, ფასი და გამყიდველის ავატარი/კალათა მობილურ ეკრანებზე ერთმანეთს ეფარებოდა, ხოლო საერთო კონტეინერს აკლდა ჰაერი (Padding).
* **გადაწყვეტა:**
  * **ქარდის კონტეინერი:** მიიღო `p-4 sm:p-6 flex-1 flex flex-col relative space-y-3`, რამაც მისცა დამატებითი ვერტიკალური და ჰორიზონტალური სივრცე ელემენტებს ასასუნთქად.
  * **სათაური და ტექსტი:** სატიტულე ნაწილს დაედო `line-clamp-1` და `mb-0.5 sm:mb-1`, ხოლო აღწერას — `mb-2 sm:mb-3` და `line-clamp-2`, რითაც შეიზღუდა ტექსტის გაწელვა და დაცული იქნა ქარდების თანაბარი სიმაღლე.
  * **ფასი და ტიპის ბეიჯი:** კონტეინერს დაემატა `flex flex-wrap items-center justify-between gap-2 mb-3`. ფასის ინფორმაციამ მიიღო `min-w-0`, ხოლო ბეიჯს დაედო `shrink-0`, რაც გამორიცხავს მათ ფიზიკურ გადაფარვას და მცირე ეკრანზე ავტომატურად გადააქვს ელემენტები ახალ ხაზზე.
  * **ავატარი და ნავიგაცია:** გარე კონტეინერს დაემატა `flex-wrap xs:flex-nowrap`, ხოლო ჩატისა და კალათის სექციას — `ml-auto shrink-0`, რამაც იდეალურად დააშორა გამყიდველის სახელი/ავატარი და ფუნქციური ღილაკები ერთმანეთს.
